# Admin Flows

## Merchant Approval

Expected operational flow:

1. customer/merchant user submits merchant application from desktop
2. backend stores application in YummyDoors
3. admin views queue in merchant applications screen
4. admin approves or rejects
5. approved user gets merchant workspace / restaurant context in desktop

## Ingestion

Admin should be able to create or manage:

- restaurants
- categories
- menu items
- promos
- reservation-related operational data

This is the bootstrap path before all merchant self-service is complete.
